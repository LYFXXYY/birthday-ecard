# 密码安全机制说明

## 一、加密算法

项目使用 **bcryptjs** 库进行密码加密，这是业界标准的密码哈希算法。

### 1.1 加密流程

```javascript
// server/src/utils/password.js
import bcrypt from 'bcryptjs';

// 密码加密
export const hashPassword = async (password) => {
  const salt = await bcrypt.genSalt(10);  // 生成盐值，rounds=10
  return bcrypt.hash(password, salt);      // 哈希密码
};

// 密码验证
export const comparePassword = async (password, hash) => {
  return bcrypt.compare(password, hash);
};
```

### 1.2 技术细节

| 参数 | 值 | 说明 |
|------|-----|------|
| 算法 | bcrypt | 基于 Blowfish 密码的自适应哈希函数 |
| Salt Rounds | 10 | 计算复杂度因子，2^10 = 1024 次迭代 |
| 盐值 | 自动生成 | 每次加密随机生成 16 字节盐值 |
| 输出长度 | 60 字符 | 固定格式：`$2a$10$...` |

### 1.3 哈希值格式

```
$2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAgcfl7p92ldGxad68LJZdL17lhWy
│   │  │
│   │  └─ 哈希值（53字符）
│   └──── Salt Rounds（10）
──────── 算法版本（2a）
```

## 二、存储方式

### 2.1 数据库字段

```sql
-- Admin 模型
password_hash VARCHAR(255) NOT NULL
password_changed_at DATETIME NOT NULL
must_change_password BOOLEAN DEFAULT TRUE
```

### 2.2 存储特点

- **明文永不存储**：数据库中只保存哈希值，无法逆向还原
- **盐值内置**：bcrypt 将盐值嵌入哈希值中，无需单独存储
- **固定长度**：无论原始密码多长，哈希值始终 60 字符

## 三、安全特性

### 3.1 防暴力破解

| 攻击方式 | 防护措施 |
|---------|---------|
| 彩虹表攻击 | 每次加密随机盐值，相同密码产生不同哈希 |
| 暴力破解 | Salt Rounds=10，单次哈希约 100ms，大幅降低破解速度 |
| 字典攻击 | 密码复杂度策略（见下文） |

### 3.2 密码复杂度策略

```javascript
// server/src/utils/passwordValidator.js
export const validatePassword = (password) => {
  if (password.length < 8) return { valid: false, message: '密码至少8位' };
  if (!/[A-Z]/.test(password)) return { valid: false, message: '需包含大写字母' };
  if (!/[a-z]/.test(password)) return { valid: false, message: '需包含小写字母' };
  if (!/[0-9]/.test(password)) return { valid: false, message: '需包含数字' };
  return { valid: true };
};
```

**要求：**
- 最少 8 位
- 包含大写字母
- 包含小写字母
- 包含数字

### 3.3 密码过期机制

```javascript
// 登录时检查密码年龄
const passwordAge = now - new Date(admin.password_changed_at);
const ninetyDaysMs = 90 * 24 * 60 * 60 * 1000;
const passwordExpired = passwordAge > ninetyDaysMs;
```

- 密码有效期：90 天
- 过期后强制要求修改
- 首次登录强制修改密码（`must_change_password = true`）

## 四、会话安全

### 4.1 JWT Token

```javascript
// server/src/utils/jwt.js
import jwt from 'jsonwebtoken';

export const generateToken = (payload) => {
  return jwt.sign(payload, config.jwtSecret, {
    expiresIn: config.jwtExpiresIn  // 默认 24h
  });
};
```

**特点：**
- 签名密钥：`JWT_SECRET`（.env 中配置，生产环境必须为随机长字符串）
- 有效期：24 小时
- 载荷：用户 ID、用户名、显示名

### 4.2 会话管理

```sql
-- ActiveSession 表
admin_id INT
token VARCHAR(500)
ip_address VARCHAR(45)
user_agent VARCHAR(255)
last_activity DATETIME
expires_at DATETIME
```

- 每次登录创建会话记录
- 并发会话数限制（默认 5 个）
- 超出限制自动删除最早会话
- 退出登录时删除会话记录

## 五、安全建议

### 5.1 生产环境配置

```env
# JWT_SECRET 必须为随机长字符串（至少 64 位）
# 生成方法：openssl rand -base64 48
JWT_SECRET=这里放随机生成的长字符串

# 缩短 Token 有效期（可选）
JWT_EXPIRES_IN=12h

# 限制并发会话数
MAX_CONCURRENT_SESSIONS=3
```

### 5.2 定期审计

- 检查 `password_changed_at` 字段，确保用户定期修改密码
- 监控 `ActiveSession` 表，发现异常登录行为
- 定期轮换 `JWT_SECRET`（需要所有用户重新登录）

### 5.3 传输安全

- 生产环境必须使用 HTTPS
- Nginx 配置 SSL 证书
- 启用 HSTS 头

## 六、与其他方案对比

| 方案 | 安全性 | 性能 | 推荐度 |
|------|-------|------|--------|
| **bcrypt**（本项目） | ★★★★★ | 中等（100ms/次） | ✅ 推荐 |
| argon2 | ★★★★★ | 中等 | ✅ 可选（更新标准） |
| PBKDF2 | ★★★★ | 较快 | ⚠️ 可用 |
| SHA-256（无盐） | ★ | 极快 | ❌ 不推荐 |
| MD5 |  | 极快 | ❌ 已破解 |

**结论：** bcrypt 是当前最成熟、最广泛使用的密码哈希方案，完全满足企业级安全需求。
