## 1. 项目启动详细教程

### 1.1 环境准备

在开始之前，确保电脑已安装以下软件：

| 软件 | 最低版本 | 用途 | 验证命令 |
|------|----------|------|----------|
| Node.js | 18.x+ | JavaScript 运行时 | `node -v` |
| npm | 9.x+ | 包管理器（随 Node.js 安装） | `npm -v` |
| MySQL | 8.0+ | 关系型数据库 | `mysql --version` |
| IntelliJ IDEA | 2023.x+ | 开发工具（或其他 IDE） | - |
| 浏览器 | Chrome / Edge | 前端访问 | - |

> 在终端输入验证命令检查是否已安装。如未安装，先完成安装再继续。

### 1.2 创建 MySQL 数据库

1. 打开 MySQL 客户端（命令行或 Navicat 等工具）
2. 执行以下 SQL：

```sql
CREATE DATABASE birthday DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> 不需要手动建表。后端启动时 Sequelize ORM 会自动创建所有数据表。

### 1.3 用 IDEA 打开项目

1. 打开 IntelliJ IDEA
2. 选择 **File → Open**，选择 `birthday-card-system` 文件夹
3. 等待 IDEA 索引完成（右下角进度条消失）

### 1.4 配置后端环境变量

1. 在 IDEA 左侧项目树中，展开 `server/` 目录
2. 复制 `.env.example` 文件，重命名为 `.env`
3. 双击打开 `.env`，修改以下配置：

```env
# [必填] 改成你的 MySQL 密码
DB_PASSWORD=你的MySQL密码

# [必填] 改成一个随机字符串（至少32个字符）
JWT_SECRET=改成你自己的随机密钥字符串至少32个字符
```

#### 可自定义的配置项

| 变量名 | 默认值 | 说明 | 何时需要修改 |
|--------|--------|------|-------------|
| `DB_HOST` | localhost | MySQL 地址 | 数据库不在本机时 |
| `DB_PORT` | 3306 | MySQL 端口 | MySQL 使用非默认端口时 |
| `DB_NAME` | birthday | 数据库名 | 使用了不同的数据库名时 |
| `DB_USER` | root | 数据库用户 | 使用非 root 用户时 |
| `DB_PASSWORD` | (必填) | 数据库密码 | **必须修改** |
| `JWT_SECRET` | (必填) | JWT 签名密钥 | **必须修改为随机长字符串** |
| `JWT_EXPIRES_IN` | 24h | Token 过期时间 | 需要调整登录有效期时 |
| `PORT` | 3000 | 后端服务端口 | 端口 3000 被占用时 |
| `NODE_ENV` | development | 运行环境 | 部署生产时改为 production |
| `BASE_URL` | http://localhost:3000 | 贺卡访问基础 URL | **部署后必须改为实际域名** |
| `SENDER_NAME` | 公司工会 | 贺卡发送者名称 | 需要自定义发送者名称时 |
| `COMPANY_NAME` | 公司工会 | 公司名称（用于 `{{company}}` 占位符） | 需要显示公司名称时 |
| `LOGO_URL` | (空) | 公司 Logo 图片 URL | 需要在贺卡中显示 Logo 时 |
| `SMS_PROVIDER` | mock | 短信发送模式（mock/carrier） | 对接真实运营商时改为 carrier |

#### CSP 5G 视频短信配置（生产环境）

对接中国移动运营商时需要配置以下变量，也可通过前端"短信配置"页面在线编辑：

| 变量名 | 说明 |
|--------|------|
| `CSP_APP_ID` | ChatbotID |
| `CSP_PASSWORD` | App 密码 |
| `CSP_SERVER_ROOT` | API 根地址（默认 `https://api.5gcsp.mas.10086.cn/ocsp/developer`） |
| `CSP_FILE_SERVER_ROOT` | 媒体文件服务地址 |
| `CSP_CHATBOT_URI` | Chatbot URI |
| `CSP_CALLBACK_URL` | 业务回调地址（需公网可访问） |
| `CSP_VIDEO_TEMPLATE_ID` | 5G 视频模板 ID |
| `CSP_TEMP_STORE_TIME` | 离线缓存时长（秒），默认 259200（3 天） |

> 配置变更后即时生效，无需重启服务。每次发送时系统自动从 `.env` 文件重新加载配置。

#### 前端环境变量

前端环境变量在 `admin-web/.env.development` 中配置：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `VITE_API_BASE_URL` | /api | API 基础路径（通过 Vite 代理转发） |
| `VITE_APP_TITLE` | 生日贺卡管理系统（开发环境） | 浏览器标签标题 |
| `VITE_USE_MOCK` | (未设置) | 设为 `true` 启用模拟数据（无需后端） |

### 1.5 安装后端依赖并启动

1. 在 IDEA 底部点击 **Terminal**（终端）标签
2. 输入以下命令：

```bash
cd server
npm install
```

3. 等待安装完成（会出现 `added xxx packages` 提示）
4. 启动后端服务：

```bash
npm run dev
```

5. 看到以下输出表示**后端启动成功**：

```
服务器运行在 http://localhost:3000
数据库连接成功
默认管理员账户已初始化（admin/123456）
定时任务已启动：每天 08:00 检查生日员工
监控进程已启动
```

> **不要关闭这个终端窗口！** 后端服务需要持续运行。后端启动时会自动拉起独立监控子进程，无需单独启动监控。

### 1.6 启动前端

1. 在 IDEA 终端标签旁边，点击 **+** 号新建一个终端
2. 输入以下命令：

```bash
cd admin-web
npm install
```

3. 等待安装完成后，启动前端开发服务器：

```bash
npm run dev
```

4. 看到以下输出表示**前端启动成功**：

```
  VITE v5.4.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://xxx.xxx.xxx.xxx:5173/
```

> 前端启动后会自动打开浏览器。如果没有自动打开，手动访问 `http://localhost:5173`。

### 1.7 验证启动成功

1. 浏览器打开 `http://localhost:5173`，看到登录页面
2. 使用默认管理员账号登录：
   - **用户名**：`admin`
   - **密码**：`123456`
3. 登录成功后进入仪表盘页面，展示 4 个核心指标卡片和系统健康状态，说明前后端都已正常运行

### 1.8 日常启动流程（快速参考）

```
1. 确保 MySQL 服务正在运行
2. IDEA 打开项目
3. 终端1：cd server → npm run dev        （启动后端 + 自动启动监控）
4. 终端2：cd admin-web → npm run dev     （启动前端）
5. 浏览器访问 http://localhost:5173
```

### 1.9 停止服务

- 在 IDEA 终端中按 `Ctrl + C` 停止当前运行的服务
- 关闭 IDEA 会自动终止所有终端进程（包括监控子进程）
- 后端停止后前端仍可访问页面，但所有 API 请求会失败

### 1.10 常见问题排查

| 问题现象 | 原因 | 解决方法 |
|---------|------|----------|
| 前端白屏/空白页 | 残留 Node 进程占用端口 | 打开任务管理器，结束所有 `node.exe` 进程，重新启动 |
| `EADDRINUSE :::3000` | 后端端口被占用 | 结束 `node.exe` 进程，或修改 `.env` 中的 `PORT` |
| `EADDRINUSE :::5173` | 前端端口被占用 | 结束 `node.exe` 进程，或 `npx kill-port 5173` |
| `Access denied for user 'root'` | MySQL 密码错误 | 检查 `.env` 中 `DB_PASSWORD` |
| `Unknown database 'birthday'` | 数据库未创建 | 执行第 1.2 步的 CREATE DATABASE |
| 前端页面无内容（无报错） | 后端未启动或代理失败 | 确认后端 `npm run dev` 正在运行 |
| 登录提示"网络错误" | 后端未启动或端口不对 | 确认后端运行在 3000 端口，F12 查看 Network |
| `Cannot find module` | 依赖未安装 | 在对应目录重新 `npm install` |
| 仪表盘数据全为 0 | Mock 模式已开启 | 检查 `.env.development` 中 `VITE_USE_MOCK` 是否为 true |

> **重要提示**：如果重启后仍报端口占用，在 CMD 中执行 `taskkill /IM node.exe /F` 强制结束所有 Node 进程。

## 2. 使用教程

### 2.1 登录系统

1. 浏览器访问 `http://localhost:5173`
2. 输入用户名 `admin`，密码 `123456`
3. 点击 **登录** 按钮
4. 登录成功后自动跳转到仪表盘

> 首次登录后建议修改默认密码。点击右上角用户名 → 修改密码。

### 2.2 仪表盘

登录后默认进入仪表盘页面，展示系统运营概况。

**核心指标**：页面顶部展示 4 个统计卡片——员工总数、今日生日数量、明日生日数量、总发送数量。

**今日生日员工**：列表展示当天过生日的员工，显示姓名、部门、手机号。每位员工旁边有"测试发送"按钮，可手动触发贺卡生成和发送流程，发送过程中显示进度条，完成后变为"已发送"状态。如果当天没有人生日，显示空状态提示。

**系统健康状态**：整合了独立监控项目的核心数据，展示服务健康状态、发送统计、最后心跳时间、内存占用等信息，30 秒自动轮询刷新。

**月度发送趋势**：图表展示近一个月的每日发送数量趋势。

### 2.3 员工管理

点击左侧菜单 **人员管理** 进入。

**查看员工列表**：列表显示员工姓名、性别、生日、手机号、部门、职位等信息，支持分页浏览（每页 10 条），可按姓名/手机号搜索，按部门筛选。

**新增员工**：点击 **新增员工** 按钮，在弹出对话框中填写员工信息（姓名、性别、生日、手机号为必填），可选择是否指定默认贺卡模板。

**编辑员工**：在列表中找到目标员工，点击 **编辑** 按钮修改信息。

**删除员工**：点击 **删除** 按钮，确认永久删除（硬删除，不可恢复，同时删除该员工的所有发送记录）。

**手动发送贺卡**：在列表中找到目标员工，点击 **发送** 按钮，系统自动匹配模板并生成贺卡。

**批量导入员工**：点击 **批量导入** 按钮，可先下载 Excel 模板，按格式填写后上传（支持 `.xlsx`、`.xls`、`.csv` 格式）。

**Excel 文件列名要求**：

| 中文列名 | 英文列名 | 必填 |
|----------|----------|------|
| 姓名 | name | 是 |
| 性别 | gender | 是（male/female） |
| 生日 | birthday | 是（YYYY-MM-DD） |
| 手机号 | phone | 是 |
| 部门 | department | 否 |
| 职位 | position | 否 |

### 2.4 模板管理

点击左侧菜单 **模板管理** 进入。

**查看模板列表**：显示模板名称、适用员工等级（管理层/三级经理/普通员工/通用）、关联祝福语等信息，点击"预览"可查看贺卡效果。

> 当前系统内置 13 套贺卡模板，分为员工级（4 页）和管理级（7 页）两类。页数由模板类型决定，编辑时自动设定，无需手动配置。

**编辑模板**：点击 **编辑** 按钮，在"贺卡文本内容"编辑框中修改祝福文本（纯文本，无需 HTML 编辑），可修改模板名称、描述、适用等级（employee_level）等元数据。系统会自动将编辑内容注入到模板的可编辑区域，保留原始 HTML 结构。保存时根据所选员工等级自动设定页数（管理层/三级经理 → 7 页，普通员工 → 4 页）。

**祝福语一键补配**：模板列表提供"一键补配祝福语"按钮，点击后系统自动为缺少默认祝福语的模板随机分配匹配的祝福语。操作前会弹出确认对话框。

**预览与断句**：模板中的祝福语支持按中文逗号 `，` 和分号 `；` 自动断句（每段一行）。右侧实时预览使用 Vue `v-html` 渲染，不会执行模板内的 `<script>` 标签，因此部分 JS 驱动的视觉效果（如粒子特效、断行动画）在实时预览中不显示。如需查看完整效果，点击 **新窗口预览** 按钮，在新窗口中运行完整的模板脚本。

**删除模板**：点击 **删除** 按钮，确认删除（硬删除，关联的员工将解除默认模板绑定）。

### 2.5 发送记录

点击左侧菜单 **发送记录** 进入。

**查看记录列表**：分页展示所有发送记录，每条记录显示员工信息、短信内容、使用的模板、发送状态、收到信息状态、收到信息时间和操作。失败记录会显示错误原因。

**发送状态**：记录贺卡生成和发送的生命周期，包含以下状态：pending（待处理）→ recording（录制中）→ recorded（录制完成）→ sending（发送中）→ success（成功）/ failed（失败）。可按状态和日期范围筛选。页面顶部展示统计信息：总发送数、成功数、失败数、成功率。

**收到信息状态**：显示运营商（中国移动）的投递结果，包含 `delivery_status`（投递状态：pending/success/failed/timeout）和 `delivery_time`（投递确认时间）。该数据由运营商回调接口自动更新，当前开发阶段暂无实际数据，空值显示"—"。

**手机预览**：点击短信内容中的"打开贺卡"或操作列的"预览"按钮，弹出手机模拟器对话框，查看贺卡在手机端的实际效果。

**删除记录**：点击 **删除** 按钮，确认删除（硬删除，不可恢复）。

### 2.6 祝福语管理

点击左侧菜单 **祝福语管理** 进入。

可以创建、编辑和删除祝福语。每条祝福语可设置适用的员工等级（管理层/三级经理/普通员工/通用），系统在自动匹配时根据员工等级从对应等级的祝福语中随机选取。

> 旧版支持按性别和年龄范围匹配，该功能已在前端移除（数据库字段保留兼容旧数据），当前匹配仅依赖员工等级。

删除有模板引用的祝福语时系统会显示警告提示。

### 2.7 短信配置

点击左侧菜单 **短信配置** 进入（仅管理员可见）。

该页面用于在线管理中国移动 CSP 5G 视频短信接口（V2.4.3）的配置参数，包括 ChatbotID、App 密码、API 地址、回调地址、视频模板 ID 等。所有配置项以表单形式展示，每项标注当前配置状态（已配置/未配置），密码类字段以密文显示。

修改后点击保存，配置即时写入 `.env` 文件，下次发送时自动加载最新配置，无需重启服务。

### 2.8 贺卡访问

贺卡生成后会获得一个唯一链接，格式为：

```
http://localhost:3000/card/{cardId}
```

无需登录即可访问，贺卡页面包含动画效果和背景音乐，适配移动端浏览器（微信、短信内置浏览器）。部署生产环境后，`BASE_URL` 需改为实际域名。

### 2.9 操作日志与系统日志

**操作日志**：记录所有管理操作的审计日志（谁在什么时间做了什么），包括员工增删改、模板编辑、祝福语变更等。点击左侧菜单 **操作日志** 查看。

**系统日志**：记录后端运行时的技术日志（数据库查询、错误堆栈、定时任务执行等），用于开发调试和故障排查。点击左侧菜单 **系统日志** 查看。

## 3. 系统监控

### 3.1 架构概述

系统采用双层监控架构：

| | 主项目监控模块 | 独立监控项目 |
|---|---|---|
| 运行位置 | 主项目进程内 | 独立 Node 进程（`server/monitor/`） |
| 监控对象 | 内部组件健康（数据库、心跳、失败率） | 主项目整体存活（HTTP、端口、心跳文件） |
| 后端挂了 | 失效 | **仍能检测并告警** |
| 数据展示 | 前端首页 Dashboard | 控制台 + 日志文件 |

两者互补：主项目监控模块为前端 Dashboard 提供实时数据展示，独立监控项目从外部视角检测主项目是否存活，在后端崩溃时仍能发现问题。

### 3.2 独立监控项目

**项目位置**：`server/monitor/`（主项目内，但独立进程）

**技术栈**：Node.js ESM，独立依赖（`node-cron`、`mysql2`、`dotenv`），不依赖主项目代码。

**核心检测维度**：

| 检测项 | 检测方式 | 频率 | 判断标准 |
|--------|----------|------|----------|
| HTTP 健康检查 | `GET /api/health` | 30 秒 | 超时或非 200 为异常 |
| 端口存活检测 | TCP 连接 localhost:3000 | 60 秒 | 连接失败为异常 |
| 心跳文件监控 | 读取心跳文件时间戳 | 60 秒 | 超过 25 小时为异常 |
| 数据库直连检查 | MySQL 客户端直连 | 5 分钟 | 连接失败为异常 |

**启动方式**：后端 `app.js` 启动时自动通过 `spawn` 创建监控子进程（`detached: true`），后端正常退出时监控随之关闭；后端异常崩溃时，监控进程继续独立运行并检测异常。

**日志输出**：`server/monitor/logs/monitor-YYYY-MM-DD.log`，按日期分文件，每天凌晨 2:00 自动清理超过 30 天的日志。

也可单独启动监控：

```bash
cd server/monitor
npm start
```

### 3.3 运营商回调（SMS Callback）

系统提供两个公开接口（无需认证），用于接收中国移动的发送状态回调：

- `POST /api/sms-callback/StatusReportNotification/:chatbotUri` — 接收投递状态报告，解析 XML 消息体，更新对应发送记录的 `delivery_status` 和 `delivery_time`。无论处理成功与否均返回 HTTP 200，避免运营商重试风暴。
- `POST /api/sms-callback/InboundMessageNotification/:chatbotUri` — 预留的上行消息（员工回复）接收接口，当前仅记录日志。

> 回调地址需要公网可访问，部署时必须将 `CSP_CALLBACK_URL` 配置为实际的外部域名。

## 4. API 调试与 Postman

本系统提供完整的 RESTful API 接口，涵盖认证、员工管理、模板管理、祝福语管理、发送记录、部门管理、短信配置、系统监控等模块。所有模块均支持 DELETE 操作（数据库硬删除，级联清除关联数据）。

如需进行 API 级别的调试和测试，可以使用 **Postman** 或其他 HTTP 客户端工具直接请求后端接口（默认地址 `http://localhost:3000`）。所有接口均使用统一的 `{code, message, data}` 响应格式，除登录和健康检查外，其余接口需要在请求头中携带 JWT Token 进行认证。

详细的 API 接口列表、请求参数和响应示例可参考 [生日贺卡项目开发文档.md](生日贺卡项目开发文档.md) 的第 3 节"API 接口规范"。

## 6. 数据模型概览

系统共 9 个数据模型，核心关系如下：

| 模型 | 说明 | 关键字段 |
|------|------|----------|
| Admin | 管理员账户 | username, password(bcrypt) |
| Employee | 员工 | name, gender, birthday, phone, department, position, level, default_template_id |
| Department | 部门（三级树形结构） | name, code, level, parent_id, sort_order |
| Template | 贺卡模板 | name, employee_level, page_count, folder_path, default_blessing_id |
| Blessing | 祝福语 | content, match_employee_level, is_active |
| SendRecord | 发送记录 | employee_id, template_id, card_id, sms_content, status, delivery_status, delivery_time |
| ActiveSession | 活跃会话（并发登录控制） | user_id, token, created_at |
| OperationLog | 操作审计日志 | user, action, target, detail |
| SystemLog | 系统技术日志 | level, module, message |

**员工等级（employee_level）**：management（管理层）、manager（三级经理）、employee（普通员工），模板和祝福语按此等级匹配。

**发送状态流转**：pending → recording → recorded → sending → success / failed。视频录制失败时状态强制 failed，不被后续 SMS 成功覆盖。
