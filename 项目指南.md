## 1. 项目概述

### 1.1 系统简介

本系统是一套面向企业工会的**生日祝福自动化管理平台**。系统能够根据员工生日信息，自动生成个性化电子贺卡，并通过彩信/短信将贺卡链接发送给员工，实现生日祝福的自动化和个性化。

### 1.2 核心流程

```
录入员工信息 → 系统每日检查当天生日 → 自动匹配贺卡模板 → 生成个性化贺卡 → 发送彩信/短信
```

管理员通过 Web 管理后台维护员工信息和贺卡模板，系统在每天 08:00 自动检查当天过生日的员工，根据模板匹配规则（手动指定 > 性别+年龄 > 性别 > 通用）选择合适的贺卡模板，替换姓名、部门、祝福语等占位符后生成 HTML 贺卡文件，最后通过短信/彩信通道将贺卡链接发送给员工。

### 1.3 技术架构

项目采用**前后端分离**架构：

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 后端 | Node.js + Express 5.x | ES Modules，端口 3000 |
| 前端 | Vue 3 + Element Plus + Vite | 管理后台，端口 5173 |
| 数据库 | MySQL 8.0+ + Sequelize ORM | 自动建表，连接池管理 |
| 认证 | JWT (jsonwebtoken + bcryptjs) | Token 认证，24h 过期 |
| 定时任务 | node-cron | 每日 08:00 自动检查 |
| 文件处理 | Multer + xlsx (SheetJS) | Excel/CSV 导入 |

## 2. 功能介绍

### 2.1 仪表盘

系统首页展示关键运营数据概览：

- **员工总数** — 系统中已录入的员工数量
- **模板数量** — 可用的贺卡模板数量
- **发送统计** — 总发送数、成功数、失败数、成功率
- **今日生日** — 当天过生日的员工列表

### 2.2 员工管理

管理企业员工信息，支持单个录入和批量导入两种方式：

| 功能 | 说明 |
|------|------|
| 新增员工 | 录入姓名、性别、生日、手机号、部门、职位 |
| 编辑员工 | 修改员工信息，可指定默认贺卡模板 |
| 删除员工 | 硬删除（永久删除，同时删除该员工的所有发送记录） |
| 搜索筛选 | 按姓名/手机号关键词搜索，按部门筛选 |
| 批量导入 | 上传 Excel (.xlsx/.xls) 或 CSV 文件批量导入 |
| 手动发送 | 为指定员工手动生成贺卡并发送 |
| 生日查询 | 查看今天过生日的员工列表 |

**员工字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 (name) | 字符串 | 是 | 员工姓名，将显示在贺卡中 |
| 性别 (gender) | male/female | 是 | 用于贺卡模板匹配 |
| 生日 (birthday) | 日期 | 是 | 用于判断发送时机 |
| 手机号 (phone) | 字符串 | 是 | 彩信/短信接收号码 |
| 部门 (department) | 字符串 | 否 | 显示在贺卡中 |
| 职位 (position) | 字符串 | 否 | 员工职位 |
| 默认模板 (default_template_id) | 数字 | 否 | 手动指定贺卡模板 |

### 2.3 模板管理

贺卡 HTML 模板的编辑和预览功能（模板文件由后端 `server/src/data/` 目录管理，启动时自动入库）：

| 功能 | 说明 |
|------|------|
| 编辑模板 | 修改模板的祝福文本内容和匹配规则 |
| 删除模板 | 硬删除模板，自动解除关联员工的默认模板绑定 |
| 模板预览 | 在新窗口中查看贺卡效果（手机端模拟） |
| 绑定祝福语 | 为模板选择默认祝福语 |
| 模板列表 | 查看所有模板，显示匹配条件 |

**模板编辑方式：**

管理员在编辑页面只需修改“贺卡文本内容”（纯文本），系统会自动将编辑内容注入到模板的可编辑区域（`<!-- editable -->` 标记内），保留原始 HTML 结构不变。

**模板占位符：**

| 占位符 | 替换内容 |
|--------|----------|
| `{{name}}` | 员工姓名 |
| `{{department}}` | 员工部门 |
| `{{position}}` | 员工职位 |
| `{{birthday}}` | 生日日期（M月D日） |
| `{{sender}}` | 发送者名称（配置项） |
| `{{blessing}}` | 祝福语（来自祝福语管理或默认值） |
| `{{year}}` | 当前年份 |

**模板匹配优先级（从高到低）：**

1. 员工手动指定的 `default_template_id`
2. 性别 + 年龄区间匹配
3. 仅性别匹配
4. 通用模板（match_gender = all）

### 2.4 发送记录

查看历史发送记录、短信内容和统计数据：

| 功能 | 说明 |
|------|------|
| 记录列表 | 分页展示所有发送记录，包含员工和模板信息 |
| 短信内容 | 展示每条记录将发送的短信文本和贺卡链接 |
| 手机预览 | 点击贺卡链接弹出手机模拟器，查看手机端效果 |
| 筛选查询 | 按员工、发送状态、日期范围筛选 |
| 发送统计 | 总发送数、成功数、失败数、成功率 |
| 重发 | 对失败的记录重新发送 |
| 删除记录 | 硬删除发送记录 |

**发送状态说明：**

| 状态 | 含义 |
|------|------|
| pending | 待发送 |
| success | 发送成功 |
| failed | 发送失败（可查看错误信息） |

### 2.5 祝福语管理

管理员可以预先创建多条祝福语，模板可绑定一条默认祝福语：

| 功能 | 说明 |
|------|------|
| 新增祝福语 | 创建祝福文本，设置适用性别和年龄范围 |
| 编辑祝福语 | 修改祝福文本和匹配条件 |
| 删除祝福语 | 删除不再使用的祝福语 |
| 祝福语列表 | 查看所有祝福语，支持分页和筛选 |

### 2.6 定时任务

系统内置自动化调度器，每天 08:00（Asia/Shanghai 时区）自动执行：

1. 查询当天过生日的员工
2. 为每位员工匹配合适的贺卡模板
3. 生成个性化贺卡 HTML 文件
4. 创建待发送记录（pending 状态）
5. 发送短信（包含祝福语和贺卡链接）
6. 根据发送结果更新记录状态（success/failed）
7. 防重复发送：已发送过的员工自动跳过

### 2.7 贺卡访问

生成的贺卡通过唯一 UUID 链接访问，无需登录认证：

- 访问地址格式：`http://服务器地址/card/{cardId}`
- 贺卡为独立 HTML 页面，包含动画效果和移动端适配
- 支持在微信、短信内置浏览器中打开

## 3. 项目启动详细教程

### 3.1 环境准备

在开始之前，确保电脑已安装以下软件：

| 软件 | 最低版本 | 用途 | 验证命令 |
|------|----------|------|----------|
| Node.js | 18.x+ | JavaScript 运行时 | `node -v` |
| npm | 9.x+ | 包管理器（随 Node.js 安装） | `npm -v` |
| MySQL | 8.0+ | 关系型数据库 | `mysql --version` |
| IntelliJ IDEA | 2023.x+ | 开发工具（或其他 IDE） | - |
| 浏览器 | Chrome / Edge | 前端访问 | - |

> 在终端输入验证命令检查是否已安装。如未安装，先完成安装再继续。

### 3.2 创建 MySQL 数据库

1. 打开 MySQL 客户端（命令行或 Navicat 等工具）
2. 执行以下 SQL：

```sql
CREATE DATABASE birthday DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> 不需要手动建表。后端启动时 Sequelize ORM 会自动创建所有数据表。

### 3.3 用 IDEA 打开项目

1. 打开 IntelliJ IDEA
2. 选择 **File → Open**，选择 `birthday-card-system` 文件夹
3. 等待 IDEA 索引完成（右下角进度条消失）

### 3.4 配置后端环境变量

1. 在 IDEA 左侧项目树中，展开 `server/` 目录
2. 复制 `.env.example` 文件，重命名为 `.env`
3. 双击打开 `.env`，修改以下配置：

```env
# [必填] 改成你的 MySQL 密码
DB_PASSWORD=你的MySQL密码

# [必填] 改成一个随机字符串（至少32个字符）
JWT_SECRET=改成你自己的随机密钥字符串至少32个字符
```

#### 可自定义的配置项

| 变量名 | 默认值 | 说明 | 何时需要修改 |
|--------|--------|------|-------------|
| `DB_HOST` | localhost | MySQL 地址 | 数据库不在本机时 |
| `DB_PORT` | 3306 | MySQL 端口 | MySQL 使用非默认端口时 |
| `DB_NAME` | birthday | 数据库名 | 使用了不同的数据库名时 |
| `DB_USER` | root | 数据库用户 | 使用非 root 用户时 |
| `DB_PASSWORD` | (必填) | 数据库密码 | **必须修改** |
| `JWT_SECRET` | (必填) | JWT 签名密钥 | **必须修改为随机长字符串** |
| `JWT_EXPIRES_IN` | 24h | Token 过期时间 | 需要调整登录有效期时 |
| `PORT` | 3000 | 后端服务端口 | 端口 3000 被占用时 |
| `NODE_ENV` | development | 运行环境 | 部署生产时改为 production |
| `CARDS_DIR` | ./generated-cards | 贺卡存储目录 | 需要自定义存储路径时 |
| `UPLOADS_DIR` | ./uploads | 上传临时目录 | 需要自定义存储路径时 |
| `BASE_URL` | http://localhost:3000 | 贺卡访问基础 URL | **部署后必须改为实际域名** |
| `SENDER_NAME` | 公司工会 | 贺卡发送者名称 | 需要自定义发送者名称时 |
| `SMS_PROVIDER` | mock | 短信发送模式（mock/carrier） | 对接真实运营商时改为 carrier |
| `SMS_API_URL` | (空) | 运营商 API 地址 | 使用 carrier 模式时必填 |
| `SMS_API_KEY` | (空) | 运营商 API 密钥 | 使用 carrier 模式时必填 |
| `SMS_API_SECRET` | (空) | 运营商 API 密钥密销 | 使用 carrier 模式时必填 |
| `SMS_SENDER_ID` | (空) | 短信发送者ID | 运营商要求时填写 |
| `SMS_MAX_RETRIES` | 3 | 短信发送最大重试次数 | 需要调整重试策略时 |
| `SMS_RETRY_DELAY` | 1000 | 重试延迟基数（ms） | 需要调整重试延迟时 |
| `SMS_TIMEOUT` | 10000 | API 请求超时时间（ms） | 运营商响应较慢时调大 |

#### 前端环境变量

前端环境变量在 `admin-web/.env.development` 中配置：

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `VITE_API_BASE_URL` | /api | API 基础路径（通过 Vite 代理转发） |
| `VITE_APP_TITLE` | 生日贺卡管理系统（开发环境） | 浏览器标签标题 |
| `VITE_USE_MOCK` | (未设置) | 设为 `true` 启用模拟数据（无需后端） |

1. 浏览器打开 `http://localhost:5173`，看到登录页面
2. 使用默认管理员账号登录：
   - **用户名**：`admin`

### 3.8 日常启动流程（快速参考）

```
1. 确保 MySQL 服务正在运行
2. IDEA 打开项目
3. 终端1：cd server → npm run dev        （启动后端）
```

### 3.9 停止服务

- 在 IDEA 终端中按 `Ctrl + C` 停止当前运行的服务
- 关闭 IDEA 会自动终止所有终端进程
- 后端停止后前端仍可访问页面，但所有 API 请求会失败

### 3.10 常见问题排查

| 问题现象 | 原因 | 解决方法 |
|---------|------|----------|
| 前端白屏/空白页 | 残留 Node 进程占用端口 | 打开任务管理器，结束所有 `node.exe` 进程，重新启动 |
| `EADDRINUSE :::3000` | 后端端口被占用 | 结束 `node.exe` 进程，或修改 `.env` 中的 `PORT` |
| `EADDRINUSE :::5173` | 前端端口被占用 | 结束 `node.exe` 进程，或 `npx kill-port 5173` |
| `Access denied for user 'root'` | MySQL 密码错误 | 检查 `.env` 中 `DB_PASSWORD` |
| `Unknown database 'birthday'` | 数据库未创建 | 执行第 3.2 步的 CREATE DATABASE |
| 前端页面无内容（无报错） | 后端未启动或代理失败 | 确认后端 `npm run dev` 正在运行 |
| 登录提示"网络错误" | 后端未启动或端口不对 | 确认后端运行在 3000 端口，F12 查看 Network |
| `Cannot find module` | 依赖未安装 | 在对应目录重新 `npm install` |
| 仪表盘数据全为 0 | Mock 模式已开启 | 检查 `.env.development` 中 `VITE_USE_MOCK` 是否为 true |

> **重要提示**：如果重启后仍报端口占用，在 CMD 中执行 `taskkill /IM node.exe /F` 强制结束所有 Node 进程。

## 4. 使用教程

### 4.1 登录系统

1. 浏览器访问 `http://localhost:5173`
2. 输入用户名 `admin`，密码 `123456`
3. 点击 **登录** 按钮
4. 登录成功后自动跳转到仪表盘

> 首次登录后建议修改默认密码。点击右上角用户名 → 修改密码。

### 4.2 仪表盘

登录后默认进入仪表盘页面，展示系统运营概况：

- **统计卡片**：员工总数、模板数量、发送总数、成功率
- **今日生日**：当天过生日的员工列表，可直接点击发送贺卡
- 数据自动从数据库实时获取，无需手动刷新

### 4.3 员工管理

点击左侧菜单 **员工管理** 进入：

#### 查看员工列表
- 列表显示员工姓名、性别、生日、手机号、部门、职位等信息
- 支持分页浏览（每页 10 条）
- 使用顶部搜索框按姓名或手机号搜索
- 使用部门筛选下拉框按部门过滤

#### 新增员工
1. 点击列表上方的 **新增员工** 按钮
2. 在弹出的对话框中填写员工信息（姓名、性别、生日、手机号为必填）
3. 可选择是否指定默认贺卡模板
4. 点击 **确定** 保存

#### 编辑员工
1. 在员工列表中找到目标员工
2. 点击操作列的 **编辑** 按钮
3. 修改需要更新的信息
4. 点击 **确定** 保存

#### 删除员工
1. 点击操作列的 **删除** 按钮
2. 确认永久删除操作（硬删除，不可恢复，同时删除该员工的所有发送记录）

#### 手动发送贺卡
1. 在员工列表中找到目标员工
2. 点击操作列的 **发送** 按钮
3. 系统自动匹配模板并生成贺卡
4. 生成成功后会显示贺卡访问链接

#### 批量导入员工
1. 点击列表上方的 **批量导入** 按钮
2. 可先点击 **下载模板** 获取 Excel 模板
3. 按模板格式填写员工信息
4. 选择文件上传（支持 .xlsx、.xls、.csv 格式）
5. 系统自动解析并导入，完成后显示导入结果和错误信息

**Excel 文件列名要求：**

| 中文列名 | 英文列名 | 必填 |
|----------|----------|------|
| 姓名 | name | 是 |
| 性别 | gender | 是（male/female） |
| 生日 | birthday | 是（YYYY-MM-DD） |
| 手机号 | phone | 是 |
| 部门 | department | 否 |
| 职位 | position | 否 |

### 4.4 模板管理

点击左侧菜单 **模板管理** 进入：

#### 查看模板列表
- 显示模板名称、适用性别、年龄区间、描述等信息
- 点击“预览”可查看贺卡实际效果

#### 删除模板
1. 点击模板列表中的 **删除** 按钮
2. 确认删除操作（硬删除，关联的员工将解除默认模板绑定）

#### 编辑模板
1. 点击模板列表中的 **编辑** 按钮
2. 在“贺卡文本内容”编辑框中修改祝福文本（纯文本，无需 HTML 知识）
3. 可修改模板名称、描述、匹配条件（性别、年龄范围）
4. 可选择绑定一条默认祝福语
5. 点击 **保存** 提交

> 系统会自动将编辑内容注入到模板的可编辑区域，保留原始 HTML 结构。

#### 预览模板
1. 点击模板列表中的 **新窗口预览** 按钮
2. 系统在新窗口中打开贺卡页面
3. 使用示例数据（张三/技术部）替换占位符，展示实际效果
4. 确认贺卡在手机浏览器中的显示效果是否满意

### 4.5 发送记录

点击左侧菜单 **发送记录** 进入：

#### 查看记录列表
- 分页展示所有发送记录
- 每条记录显示员工信息、短信内容、使用的模板、发送状态、发送时间
- **短信内容**列展示将发送的短信文本和贺卡链接
- 失败记录会显示错误原因

#### 手机预览
- 点击短信内容中的“打开贺卡”或操作列的“预览”按钮
- 弹出手机模拟器对话框，查看贺卡在手机端的实际效果
- 可点击“在新标签页打开”查看完整页面

#### 筛选查询
- **按状态筛选**：pending（待发送）/ success（成功）/ failed（失败）
- **按日期范围**：设置起止日期筛选时间段

#### 发送统计
- 页面顶部展示统计信息：总发送数、成功数、失败数、成功率
- 用于评估系统运行状况

#### 删除记录
1. 点击操作列的 **删除** 按钮
2. 确认删除操作（硬删除，不可恢复）

### 4.6 祝福语管理

点击左侧菜单 **祝福语管理** 进入：

#### 查看祝福语列表
- 分页展示所有祝福语
- 显示祝福文本、适用性别、年龄范围等信息

#### 新增祝福语

2. 填写祝福文本内容
3. 可选择设置适用性别和年龄范围
4. 点击 **保存** 提交

#### 编辑祝福语
1. 点击列表中的 **编辑** 按钮
2. 修改祝福文本或匹配条件
3. 点击 **保存** 提交

### 4.7 贺卡访问

贺卡生成后会获得一个唯一链接，格式为：

```
http://localhost:3000/card/{cardId}
```

- 无需登录即可访问
- 贺卡页面包含动画效果和背景音乐
- 适配移动端浏览器（微信、短信内置浏览器）
- 部署生产环境后，`BASE_URL` 需改为实际域名

## 5. API 调试与 Postman

本系统提供了完整的 RESTful API 接口，涵盖认证、员工管理、模板管理、祝福语管理、发送记录等模块。所有模块均支持 DELETE 操作（数据库硬删除，级联清除关联数据）。

如需进行 API 级别的调试和测试，可以使用 **Postman** 或其他 HTTP 客户端工具直接请求后端接口（默认地址 `http://localhost:3000`）。所有接口均使用统一的 `{code, message, data}` 响应格式，除登录和健康检查外，其余接口需要在请求头中携带 JWT Token 进行认证。

详细的 API 接口列表、请求参数和响应示例可参考项目中的 **前端对接指南** 文档。
