## 二、已识别并修复的问题

### 2.1 P0 - 阻塞性/严重安全问题（全部已修复）

#### 问题1: SendRecord模型缺少 `admin_id` 字段
- **严重程度**: 高 (数据丢失)
- **描述**: `models/index.js` 定义了 `SendRecord.belongsTo(Admin, { foreignKey: 'admin_id' })`，且 `employees.js` 手动生成贺卡时写入 `admin_id`，但 `SendRecord` 模型定义中没有该字段，Sequelize 会静默丢弃数据
- **修复**: 在 `SendRecord.js` 模型中添加了 `admin_id` 字段定义

#### 问题2: records路由缺少认证中间件
- **严重程度**: 高 (安全漏洞)
- **描述**: 其余路由（employees, templates）都使用了 `router.use(authMiddleware)`，唯独 `records.js` 遗漏，发送记录和统计数据无需认证即可访问
- **修复**: 在 `records.js` 路由中添加 `router.use(authMiddleware)`

#### 问题3: response.js 的 `error()` 函数参数设计混乱
- **严重程度**: 高 (数据错误)
- **描述**: 旧签名 `error(res, message, code, statusCode)` 四个参数容易混淆，`records.js` 中调用 `error(res, '员工不存在', 1, 404)` 导致业务码为1而非404，`employees.js` 中 `error(res, '部分验证失败', 400, {...})` 将对象传入了statusCode位置
- **修复**: 简化为 `error(res, message, statusCode, extraData)` 三个核心参数

#### 问题4: 调度器创建失败记录时 `template_id: null`
- **严重程度**: 高 (数据库约束违反)
- **描述**: `scheduler.js` 异常处理中创建失败记录时 `template_id: null`，但模型定义 `allowNull: false`，导致数据库约束错误
- **修复**: 重构异常处理逻辑，失败时仅记录日志，不创建不完整的数据库记录

#### 问题5: 模板CRUD存在Mass Assignment漏洞
- **严重程度**: 高 (安全漏洞)
- **描述**: `Template.create(req.body)` 和 `Template.update(req.body)` 直接将请求体传递给数据库操作，攻击者可通过请求篡改任意字段（id, timestamps等）
- **修复**: 添加 `sanitizeInput()` 白名单过滤函数

#### 问题6: 贺卡路由存在路径遍历风险
- **严重程度**: 高 (安全漏洞)
- **描述**: card路由仅对cardId做了长度检查(>=10)，未防范 `../` 等路径遍历字符
- **修复**: 使用UUID正则验证格式 + `path.basename()` + `path.resolve()` 防止目录穿越

#### 问题7: Vite 代理配置三处错误（阻塞性）
- **严重程度**: 高 (开发环境无法启动)
- **描述**:
  1. 代理目标端口 `8080` 错误，后端实际运行在 `3000`
  2. 前端开发服务器端口也是 `3000`，与后端端口冲突
  3. 存在 `rewrite` 规则 `path.replace(/^\/api/, '')` 错误地剥离了 `/api` 前缀，而后端路由本身就挂载在 `/api/*`
- **位置**: `admin-web/vite.config.ts`
- **修复**:
  - 代理目标改为 `http://localhost:3000`
  - 前端端口改为 `5173`
  - 移除 `rewrite` 规则

#### 问题8: Express 路由顺序导致 `today-birthday` 不可达
- **严重程度**: 高 (功能不可用)
- **描述**: `server/src/routes/employees.js` 中 `GET /today-birthday` 路由定义在 `GET /:id` 之后，Express 会先匹配 `:id` 参数路由，将 `"today-birthday"` 当作 id 参数处理，导致该接口永远不可达
- **位置**: `server/src/routes/employees.js`
- **修复**: 将 `GET /today-birthday` 路由移到 `GET /:id` 之前

#### 问题9: FormData Content-Type 边界丢失
- **严重程度**: 高 (功能不可用)
- **描述**: `admin-web/src/api/employees.ts` 中显式设置 `Content-Type: multipart/form-data`，导致浏览器无法自动添加 `boundary` 分隔符参数，服务器无法解析 multipart 请求体
- **位置**: `admin-web/src/api/employees.ts`
- **修复**: 移除手动设置的 Content-Type 头，让浏览器自动处理（包含 boundary）

#### 问题10: Dashboard 组件硬编码模拟数据
- **严重程度**: 高 (数据不准确)
- **描述**: `Dashboard.vue` 中 `isDevMode = true` 硬编码，导致永远使用模拟数据；员工总数硬编码为 `0`；`successRate` 字段名与后端返回的 `success_rate` 不匹配
- **位置**: `admin-web/src/views/Dashboard.vue`
- **修复**:
  - `isDevMode` 改为环境变量控制 `import.meta.env.VITE_USE_MOCK === 'true'`
  - 员工总数改为调用真实 API `getEmployeeList`
  - 字段名统一为 snake_case `success_rate`

#### 问题11: 模板预览功能被 JSON 拦截器破坏
- **严重程度**: 高 (功能不可用)
- **描述**: `GET /templates/:id/preview` 返回 HTML 内容，但 axios 响应拦截器始终尝试从 `{code, message, data}` 格式解析响应，导致 HTML 被错误处理
- **位置**: `admin-web/src/api/request.ts` + `admin-web/src/api/templates.ts`
- **修复**:
  - 响应拦截器添加 Content-Type 检测：`text/html` 响应直接返回原始数据
  - 移除 `templates.ts` 中错误的类型断言 `res as unknown as string`

#### 问题12: 员工字段白名单遗漏 snake_case 形式
- **严重程度**: 高 (数据丢失)
- **描述**: 前端编辑员工时发送 `default_template_id`（snake_case），但后端 `EMPLOYEE_FIELDS` 白名单仅包含 `defaultTemplateId`（camelCase），导致该字段被白名单过滤器静默丢弃
- **位置**: `server/src/routes/employees.js`
- **修复**: 白名单同时包含两种形式，并在 `sanitizeEmployeeInput` 中统一映射为数据库字段名

### 2.2 P1 - 重要问题（全部已修复）

#### 问题13: JWT_SECRET 使用不安全的默认值
- **描述**: `config/index.js` 中 JWT_SECRET 使用默认值 `'default_secret_change_in_production'`，生产环境若未配置，所有人都可以伪造 Token

### 2.3 前端构建与运行时修复（v1.3.0 新增）

以下问题在用户实际启动前端项目时发现并修复，属于构建配置和运行时层面的缺陷。

#### 问题22: MainLayout 使用 `<slot />` 导致子路由内容不渲染（根因）
- **严重程度**: 高 (页面白屏)
- **描述**: `MainLayout.vue` 中内容区域使用 `<slot />` 而非 `<router-view />`。`<slot />` 用于父组件向子组件传递内容，而嵌套路由渲染必须使用 `<router-view />`。导致登录后侧边栏和头部正常显示，但内容区域始终空白。
- **位置**: `admin-web/src/components/MainLayout.vue`
- **修复**: `<slot />` → `<router-view />`

#### 问题23: 缺少 `vite-env.d.ts` TypeScript 声明文件
- **严重程度**: 中 (类型检查失败)
- **描述**: 项目缺少 `vite-env.d.ts`，TypeScript 无法识别 `.vue` 模块导入和 `import.meta.env` 类型
- **修复**: 创建 `src/vite-env.d.ts`，包含 `declare module '*.vue'` 和 `/// <reference types="vite/client" />`

#### 问题24: 缺少 `tailwind.config.js` 配置文件
- **严重程度**: 低 (构建警告)
- **描述**: `postcss.config.js` 引用了 `tailwindcss: {}`，但项目根目录缺少 Tailwind 配置文件，Vite 启动时警告 `content` 选项为空
- **修复**: 创建 `tailwind.config.js`，配置 `content` 扫描路径

#### 问题25: `vite.config.ts` 引用未安装的 terser
- **严重程度**: 中 (生产构建失败)
- **描述**: `build.minify` 设置为 `'terser'`，但 `terser` 未在 `package.json` 依赖中声明
- **修复**: 改为 `minify: 'esbuild'`（Vite 内置）

#### 问题26: vue-tsc v1.8.5 与 Node.js v24 不兼容
- **严重程度**: 中 (构建脚本失败)
- **描述**: `vue-tsc` v1.8.5 的 monkey-patching 逻辑无法匹配 Node.js v24 中的 TypeScript 内部结构
- **修复**: 升级 `vue-tsc` 至 `^2.0.0`，将 `build` 脚本改为 `vite build`，新增独立的 `typecheck` 脚本

#### 问题27: `tsconfig.json` 的 `ignoreDeprecations` 值无效
- **严重程度**: 低 (TypeScript 警告)
- **描述**: `ignoreDeprecations` 设置为 `"6.0"`，但 TypeScript 5.x 仅识别 `"5.0"`
- **修复**: 改为 `"ignoreDeprecations": "5.0"`
