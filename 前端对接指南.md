# 生日彩信系统 - 前端对接指南

## 1. 项目概述

这是一个工会生日彩信和电子贺卡系统，项目名称为 `birthday-card-system`。后端采用 Node.js + Express 技术栈，前端使用 Vue 3 + Element Plus。系统主要功能包括员工管理、贺卡模板管理、自动发送生日祝福等。

**项目目录结构**：
```
birthday-card-system/
├── server/           # 后端项目（当前已完成）
├── admin-web/        # 前端项目（待创建）
└── 技术文档/
```

## 2. 后端 API 信息

### 2.1 基础配置
- **API 地址**: `http://localhost:3000`
- **认证方式**: JWT (JSON Web Token)
- **默认管理员账号**: 
  - 用户名：`admin`
  - 密码：`123456`（首次启动自动创建，请及时修改）

### 2.2 统一响应格式
所有 API 接口采用统一的 JSON 响应格式：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 2.3 认证机制
- 登录接口不需要认证，其他所有管理接口都需要 JWT 认证
- 认证头格式：`Authorization: Bearer <token>`
- Token 有效期：24 小时

## 3. API 接口详情

### 3.1 认证接口

#### 管理员登录
- **URL**: `/api/auth/login`
- **Method**: POST
- **Body**:
```json
{
  "username": "admin",
  "password": "123456789"
}
```
- **成功响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "<JWT_TOKEN>",
    "user": {
      "id": 1,
      "username": "admin",
      "display_name": "管理员"
    }
  }
}
```

#### 获取当前用户信息
- **URL**: `/api/auth/profile`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`

#### 修改密码
- **URL**: `/api/auth/change-password`
- **Method**: POST
- **Headers**: `Authorization: Bearer <token>`
- **Body**:
```json
{
  "oldPassword": "原密码",
  "newPassword": "新密码"
}
```

### 3.2 员工管理接口

#### 获取员工列表
- **URL**: `/api/employees?page=1&pageSize=10&keyword=张&department=技术部`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`
- **查询参数**:
  - `page`: 页码（默认 1）
  - `pageSize`: 每页数量（默认 20）
  - `keyword`: 搜索关键词（姓名或手机号）
  - `department`: 部门筛选

#### 新增员工
- **URL**: `/api/employees`
- **Method**: POST
- **Headers**: `Authorization: Bearer <token>`
- **Body**:
```json
{
  "name": "张三",
  "gender": "male",
  "birthday": "1990-05-15",
  "phone": "13800138000",
  "department": "技术部",
  "position": "工程师"
}
```

#### 获取员工详情
- **URL**: `/api/employees/:id`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`

#### 修改员工信息
- **URL**: `/api/employees/:id`
- **Method**: PUT
- **Headers**: `Authorization: Bearer <token>`
- **Body**: (只包含需要修改的字段)
```json
{
  "position": "高级工程师",
  "department": "研发部"
}
```

#### 删除员工（软删除）
- **URL**: `/api/employees/:id`
- **Method**: DELETE
- **Headers**: `Authorization: Bearer <token>`

#### Excel 批量导入
- **URL**: `/api/employees/import`
- **Method**: POST
- **Headers**: `Authorization: Bearer <token>`
- **Content-Type**: `multipart/form-data`
- **Body**: 文件字段名为 `file`

#### 获取今天生日的员工
- **URL**: `/api/employees/today-birthday`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`

#### 手动生成员工贺卡
- **URL**: `/api/employees/:id/generate-card`
- **Method**: POST
- **Headers**: `Authorization: Bearer <token>`

### 3.4 发送记录接口

#### 获取发送记录列表
- **URL**: `/api/records?page=1&pageSize=10&employeeId=1&status=success&startDate=2026-01-01&endDate=2026-12-31`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`

  - `pageSize`: 每页数量

  - `status`: 发送状态筛选（pending/success/failed）

#### 获取发送统计
- **URL**: `/api/records/stats`
- **Method**: GET
- **Headers**: `Authorization: Bearer <token>`

#### 测试发送（开发调试用）
- **URL**: `/api/records/test-send/:employeeId`
- **Method**: POST
- **Headers**: `Authorization: Bearer <token>`

### 3.5 贺卡公开访问接口

#### 访问生成的贺卡
- **URL**: `/card/:cardId`
- **Method**: GET
- **认证**: 无需认证
- **响应**: 渲染好的 HTML 页面

1. 在与 `server` 目录同级的位置创建 `admin-web` 目录
2. 进入 `admin-web` 目录，执行 `npm init vue@latest` 创建 Vue 3 项目

4. 安装 Pinia：`npm install pinia`
5. 安装 axios：`npm install axios`

### 4.6 关键实现要点

#### 4.6.1 axios 配置
```javascript
// src/api/request.js
import axios from 'axios';
import { useUserStore } from '@/stores/user';

const request = axios.create({
  baseURL: 'http://localhost:3000',
  timeout: 10000
});

// 请求拦截器
request.interceptors.request.use(config => {
  const userStore = useUserStore();
  if (userStore.token) {
    config.headers.Authorization = `Bearer ${userStore.token}`;
  }
  return config;
});

// 响应拦截器
request.interceptors.response.use(
  response => {
    // 统一处理响应
    if (response.data.code === 200) {
      return response.data.data;
    } else {
      // 处理业务错误
      throw new Error(response.data.message);
    }
  },
  error => {
    // 处理网络错误
    if (error.response?.status === 401) {
      // token 过期，跳转登录
      // ...
    }
    return Promise.reject(error);
  }
);

export default request;
```

#### 4.6.2 认证状态管理
```javascript
// src/stores/user.js
import { defineStore } from 'pinia';

export const useUserStore = defineStore('user', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    userInfo: JSON.parse(localStorage.getItem('userInfo') || '{}')
  }),
  actions: {
    setToken(token, userInfo) {
      this.token = token;
      this.userInfo = userInfo;
      localStorage.setItem('token', token);
      localStorage.setItem('userInfo', JSON.stringify(userInfo));
    },
    clearToken() {
      this.token = '';
      this.userInfo = {};
      localStorage.removeItem('token');
      localStorage.removeItem('userInfo');
    }
  }
});
```

#### 4.6.3 路由守卫
```javascript
// src/router/index.js
router.beforeEach((to, from, next) => {
  const userStore = useUserStore();
  if (to.path !== '/login' && !userStore.token) {
    next('/login');
  } else {
    next();
  }
});
```

## 7. 注意事项

1. **响应格式**: 所有 API 响应遵循 `{code, message, data}` 格式
2. **认证机制**: 除登录和贺卡访问接口外，其他接口都需要 JWT 认证
3. **错误处理**: 前端需统一处理 401（未认证）、403（无权限）、404（资源不存在）等错误
4. **数据安全**: 避免在前端暴露敏感信息
5. **软删除**: 删除员工实际上是将 `is_active` 字段设为 false，数据仍然保留在数据库中
