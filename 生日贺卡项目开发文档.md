# 工会生日彩信和电子贺卡系统 - 技术设计文档

## Context（背景）

工会需要在员工生日当天自动发送包含电子贺卡链接的彩信/短信。本文档为完整的技术设计文档，包含系统架构、数据库设计、API规范、前端开发指南和贺卡模板设计等内容，用于指导前后端协作开发。


**责任分工**：
- 后端开发：负责API实现、数据库设计、定时任务、贺卡生成逻辑等
- 前端开发：负责管理后台界面、贺卡展示页面实现

---

## 技术选型（已确认）

| 层面 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Node.js + Express | 学习曲线平缓，前后端语言统一 |
| 前端框架 | Vue 3 + Element Plus | 中文文档完善，适合后台管理系统 |
| 数据库 | MySQL | 结构化数据管理，学习资料丰富 |
| ORM | Sequelize | Node.js 主流 ORM，减少手写 SQL |
| 定时任务 | node-cron | 轻量级，Windows 兼容 |
| 认证 | JWT (jsonwebtoken + bcryptjs) | 简单账号密码 + token 鉴权 |
| Excel导入 | xlsx (SheetJS) | 批量导入员工数据 |
| 部署 | Windows 服务器 | 定时任务用 node-cron，无需 cron 服务 |

## 2. 数据库设计

### 2.1 ER关系图

```
Admin (1) ──── (N) SendRecord
Employee (1) ──── (N) SendRecord
Template (1) ──── (N) SendRecord
Template (1) ──── (N) Employee (default_template_id)
```

### 2.2 表结构详细设计

#### admins 表（管理员）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt加密密码 |
| display_name | VARCHAR(50) | NOT NULL | 显示名称 |
| created_at | DATETIME | DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | ON UPDATE NOW() | 更新时间 |

#### employees 表（员工）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(50) | NOT NULL | 员工姓名 |
| gender | ENUM('male','female') | NOT NULL | 性别 |
| birthday | DATE | NOT NULL | 生日（月-日用于匹配） |
| phone | VARCHAR(20) | NOT NULL | 手机号码 |
| department | VARCHAR(100) | | 部门 |
| position | VARCHAR(100) | | 职位 |
| default_template_id | INT | FK -> templates.id, NULL | 管理员手动指定的默认模板 |
| is_active | TINYINT(1) | DEFAULT 1 | 是否启用（离职可关闭） |
| created_at | DATETIME | DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | ON UPDATE NOW() | 更新时间 |

**索引**：`birthday` 字段建索引，加速每日生日查询。

#### templates 表（贺卡模板）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(100) | NOT NULL | 模板名称（如"年轻女性-清新风"） |
| description | VARCHAR(255) | | 模板描述 |
| match_gender | ENUM('male','female','all') | DEFAULT 'all' | 匹配性别 |
| match_age_min | INT | NULL | 匹配最小年龄 |
| match_age_max | INT | NULL | 匹配最大年龄 |
| match_interests | VARCHAR(255) | NULL | 匹配兴趣标签（逗号分隔） |
| html_content | TEXT | NOT NULL | 贺卡HTML模板内容 |
| preview_image | VARCHAR(255) | NULL | 模板预览图路径 |
| is_active | TINYINT(1) | DEFAULT 1 | 是否启用 |
| created_at | DATETIME | DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | ON UPDATE NOW() | 更新时间 |

**模板HTML中使用占位符**：`{{name}}`、`{{department}}`、`{{blessing}}` 等，生成时替换为实际数据。

#### send_records 表（发送记录）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| employee_id | INT | FK -> employees.id, NOT NULL | 关联员工 |
| template_id | INT | FK -> templates.id, NOT NULL | 使用的模板 |
| card_url | VARCHAR(500) | NOT NULL | 生成的贺卡访问链接 |
| send_status | ENUM('pending','success','failed') | DEFAULT 'pending' | 发送状态 |
| send_time | DATETIME | | 实际发送时间 |
| error_message | TEXT | NULL | 失败原因 |
| created_at | DATETIME | DEFAULT NOW() | 记录创建时间 |

**索引**：`employee_id` + `created_at` 联合索引，按员工和时间查询。

## 3. API接口规范

### 3.1 统一响应格式

所有API接口采用统一的JSON响应格式，包含状态码、消息和数据字段。

### 3.2 认证接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/login` | 否 | 管理员登录 |
| POST | `/api/auth/change-password` | 是 | 修改密码 |
| GET | `/api/auth/profile` | 是 | 获取当前管理员信息 |

### 3.3 员工管理接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/employees` | 是 | 获取员工列表（分页、搜索） |
| GET | `/api/employees/:id` | 是 | 获取单个员工详情 |
| POST | `/api/employees` | 是 | 新增员工 |
| PUT | `/api/employees/:id` | 是 | 修改员工信息 |
| DELETE | `/api/employees/:id` | 是 | 删除员工 |
| POST | `/api/employees/import` | 是 | Excel批量导入 |
| GET | `/api/employees/today-birthday` | 是 | 获取今天生日的员工 |

### 3.4 贺卡模板管理接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/templates` | 是 | 获取模板列表 |
| GET | `/api/templates/:id` | 是 | 获取模板详情（含HTML内容） |
| POST | `/api/templates` | 是 | 新增模板 |
| PUT | `/api/templates/:id` | 是 | 修改模板 |
| DELETE | `/api/templates/:id` | 是 | 删除模板 |
| GET | `/api/templates/:id/preview` | 是 | 预览模板渲染效果 |

### 3.5 发送记录接口

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/records` | 是 | 获取发送记录（分页、筛选） |
| GET | `/api/records/stats` | 是 | 获取统计数据（总发送数、成功率等） |
| POST | `/api/records/test-send/:employeeId` | 是 | 手动测试发送（开发调试用） |

### 3.6 贺卡公开访问接口（无需认证）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/card/:cardId` | 否 | 访问生成的贺卡页面 |

- `cardId` 为发送记录中生成的唯一标识
- 直接返回渲染好的 HTML 页面
- 贺卡页面需适配手机浏览器（响应式设计）

## 4. 核心业务流程

### 4.1 每日定时任务流程

```
每天 08:00 触发 node-cron
    │
    ▼
查询 birthday 月-日 = 今天的员工（is_active = 1）
    │
    ▼
对每个今天生日的员工：
    │
    ├─ 1. 确定模板
    │     ├─ 如果员工有 default_template_id → 使用指定模板
    │     └─ 否则 → 调用 templateMatcher 根据性别、年龄自动匹配
    │
    ├─ 2. 生成贺卡
    │     ├─ 读取模板 HTML 内容
    │     ├─ 替换占位符 {{name}}, {{department}}, {{blessing}} 等
    │     ├─ 生成唯一 cardId
    │     └─ 保存 HTML 文件到 generated-cards/ 目录
    │
    ├─ 3. 构建发送内容
    │     └─ "亲爱的{name}，祝您生日快乐！点击查看您的专属贺卡：{cardUrl}"
    │
    ├─ 4. 调用短信发送接口
    │     ├─ 成功 → send_records 记录 status = 'success'
    │     └─ 失败 → send_records 记录 status = 'failed' + error_message
    │
    └─ 5. 处理下一个员工
```

### 4.2 模板匹配逻辑

根据员工信息自动匹配最佳模板，优先级为：手动指定 > 性别+年龄 > 性别 > 通用模板。

### 4.3 贺卡生成逻辑

根据模板和员工信息生成个性化贺卡，替换占位符并保存为独立HTML文件。

## 5. 前端开发指南

### 5.1 技术栈和配置

- **框架**：Vue 3 + Element Plus
- **构建工具**：Vite
- **状态管理**：Pinia
- **HTTP客户端**：axios
- **路由**：Vue Router
- **环境配置**：.env 文件管理

### 5.2 项目结构

```
admin-web/
├── src/
│   ├── api/           # API请求封装
│   ├── views/         # 页面组件
│   ├── components/    # 通用组件
│   ├── router/        # 路由配置
│   ├── stores/        # 状态管理
│   └── utils/         # 工具函数
├── public/           # 静态资源
└── package.json
```

### 5.3 路由和页面结构

| 页面 | 路由 | 功能 |
|------|------|------|
| 登录页 | `/login` | 管理员登录 |
| 仪表盘 | `/` | 统计概览：员工总数、今日生日数、发送成功率 |
| 员工列表 | `/employees` | 表格展示，支持搜索、分页、删除 |
| 员工录入 | `/employees/add` | 表单录入单个员工 |
| 员工编辑 | `/employees/:id/edit` | 编辑员工信息 |
| 批量导入 | `/employees/import` | 上传Excel文件，显示导入结果 |
| 模板管理 | `/templates` | 卡片式展示所有模板 |
| 模板编辑 | `/templates/:id/edit` | 编辑模板HTML，实时预览 |
| 发送记录 | `/records` | 表格展示发送历史，筛选状态 |

### 5.4 状态管理

- **用户状态**：使用 Pinia 管理，包含 token 和用户信息
- **全局状态**：可考虑使用 Pinia 管理应用级状态
- **API状态**：每个模块独立管理自己的API状态

### 5.5 API调用规范

前端使用axios作为HTTP客户端，配置请求拦截器添加JWT认证token，统一处理响应和错误。

### 5.6 组件开发要点

1. **响应式设计**：所有页面需适配不同屏幕尺寸
2. **表单验证**：使用 Element Plus Form 组件进行表单验证
3. **数据加载**：使用 loading 状态提示用户
4. **错误处理**：统一处理 API 错误并显示友好提示
5. **权限控制**：路由守卫控制页面访问权限

### 5.7 贺卡展示页面

- **技术要求**：纯HTML/CSS/JS实现，不依赖Vue
- **响应式**：优先适配手机浏览器
- **动画效果**：使用CSS动画实现流畅效果
- **占位符替换**：后端生成时替换 `{{name}}`、`{{department}}` 等占位符
- **访问方式**：通过 `/card/:cardId` URL 直接访问

## 8. 贺卡模板设计

### 8.1 模板结构分析

分析 `birthday_h5_rebuild.html` 发现这是一个两页设计的贺卡模板：
- 第一页：生日祝福主页面，包含动画效果

### 8.2 技术特性

- **纯HTML/CSS/JS实现**：无需外部依赖，便于部署
- **CSS动画**：使用CSS3动画实现流畅的视觉效果
- **SVG图形**：使用SVG绘制贺卡装饰元素
- **响应式设计**：适配不同屏幕尺寸
- **占位符系统**：使用 `{{name}}`、`{{department}}` 等占位符

### 8.3 适配要求

1. **占位符替换**：在生成贺卡时替换为实际员工信息
2. **音乐播放**：支持背景音乐（可选）
3. **图片资源**：确保所有图片资源可访问
4. **动画控制**：保持动画流畅性，避免性能问题
5. **移动端适配**：确保在手机浏览器上正常显示
